# エンジン非依存型 当たり判定デバッグ・解析ツール仕様書

## 1. 概要

本ツールはゲームエンジンに依存せず、当たり判定に関するデータを収集・解析するためのデバッグツールである。

ゲーム内で発生した当たり判定情報を記録し、Happy（期待する結果）とUnhappy（期待しない結果）の観点から分析を行う。

また、自動テストや失敗ケースの収集を行い、問題箇所の特定や改善案の提示を支援する。

---

# 2. 開発目的

ゲーム開発では当たり判定に関する不具合の調査に多くの時間を要する。

例

* 当たるべきなのに当たらない
* 当たってはいけないのに当たる
* 高速移動時に判定が抜ける
* 判定サイズが適切でない

本ツールはこれらの問題を自動的に記録・分析し、デバッグ効率を向上させることを目的とする。

---

# 3. 設計方針

本ツールはゲームを直接操作しない。

代わりにゲームから取得したデータを利用して解析を行う。

```text
ゲーム
 ↓
データ収集
 ↓
ログ保存
 ↓
解析ツール
 ↓
結果表示
```

これにより

* Unity
* Unreal Engine
* 独自エンジン

など様々な環境で利用可能とする。

# 4. システム構成

## Recorder（記録機能）

ゲーム中の情報を収集する。

### 取得対象

* Position
* Rotation
* Velocity
* Collider情報
* Collision結果

### 記録単位

フレーム単位

```text

Frame 1
Frame 2
...
```

---

## Analyzer（解析機能）

記録データを読み込み解析する。

### 解析内容

* Happy判定
* Unhappy判定
* 発生回数
* 発生率
* 発生フレーム

---

## Optimizer（改善提案機能）

記録データの解析結果や警告、自動テスト（Fuzzer）の結果を総合して改善案を提示する。

主な分析対象と提案内容:
* **誤検出（意図しない衝突）**: Colliderサイズの縮小
* **判定抜け（当たるべきで当たらない）**: Colliderサイズの拡大、CCDやRaycastの導入
* **ガタつき（Jitter）の検知**: 物理マテリアルの反発係数低下、ヒステリシスの導入
* **境界付近のゆらぎ（Fuzzing）**: 判定マージン（Skin Widthなど）の見直し

# 10. データ保存

記録結果や解析結果を、人が直感的に読めるよう表形式（Markdown）に整理し、一つのファイル（`YYYYMMDD_HHMMSS_Report.md`）として `HapiColi/Log/` に保存する。
ファイルには、**テストサマリー**、**衝突期間（Collision Periods）**、**テストの詳細（すべてのフレームの結果、オブジェクト間の距離や位置情報などを含むテーブル）**、および**ファジング結果（Fuzzing Results）**が出力される。

例:
```markdown

## 1. Summary
- **Total Tests:** 1000
- **Happy (Success):** 970
- **Unhappy (Failed):** 30
- **Success Rate:** 97.0%

## 2. Collision Periods
| Object A | Object B | Start Frame | End Frame |
|---|---|---|---|
| Player | Ground | 100 | 250 |

## 3. Test Details
| Frame | Status | Expected | Actual | Subject Pos | Target Pos | Distance | Reason |
|---|---|---|---|---|---|---|---|
| 523 | Unhappy | No Hit Wall | Hit Wall | Player (10.0, 0.0, 5.0) | Wall (10.0, 0.0, 5.0) | 0.00 | Unintended collision |
| 524 | Happy | Hit Wall | Hit Wall | Player (10.1, 0.0, 5.1) | Wall (10.0, 0.0, 5.0) | 0.14 | Success |

## 4. Fuzzing Results
| Test Name | Status | Object A Pos | Object B Pos | Description |
|---|---|---|---|---|
| Player-Wall MicroOffset | Bug Caught | Player (10.0, -0.01, 5.0) | Wall (10.0, 0.0, 5.0) | Unexpected collision detected! |
```

# 14. 改善案生成

解析結果、アナライザーによる警告、および自動テスト（ファジング）の結果から、総合的な改善案を生成する。

## ケース1：誤検出が多い

**原因推測**: 意図しない衝突判定が多数記録されている。
**提案**:
* コライダーを縮小してください。

---

## ケース2：判定抜けが多い、または高速衝突が検知された

**原因推測**: 対象に当たるべき状況で当たっていない、またはすり抜けリスクのある高速移動が警告されている。
**提案**:
* コライダーサイズを拡大してください。
* 連続衝突検知(CCD)を有効にしてください。
* 高速な物体にはレイキャストを使用してください。

---

## ケース3：ガタつき（Jitter）の検知

**原因推測**: 同一オブジェクト間で毎フレーム衝突と非衝突が切り替わる不安定な状態（ガタつき）が警告されている。
**提案**:
* 物理マテリアルの反発係数（Bounciness）を下げてください。
* 押し出し処理（Penetration Resolution）にヒステリシスを持たせてください。

---

## ケース4：ファジングテストでの問題検出

**原因推測**: 自動テスト（微小ゆらぎテスト等）により、境界値付近での判定結果の変動が検出されている。
**提案**:
* ゆらぎで判定が変わるため、境界値付近での判定マージン（Skin Widthなど）を見直してください。

# 17. 使い方（利用マニュアル）

本ツールを実際のゲームエンジン（C++環境）に組み込み、利用するための手順を以下に示す。

## 17.1. エンジンへの組み込み手順

HapiColiはエンジン非依存のため、ゲームループに少数のAPI呼び出しを追加するだけで動作する。

### 17.1.1. 初期化と更新
ゲームの初期化処理と、毎フレームの描画処理（ImGuiの描画タイミング）で以下を呼び出す。

```cpp
#include "HapiColi/HapiColi.h"

// 初期化時
HapiColi::HapiColi::GetInstance().Initialize();

// 毎フレーム（ImGui描画中）
HapiColi::HapiColi::GetInstance().Update();
```

### 17.1.2. データの記録
フレームの開始時、各オブジェクトの更新・判定後、フレームの終了時にAPIを呼び出してデータを送信する。

```cpp
// フレームの最初
HapiColi::HapiColi::GetInstance().BeginFrame(deltaTime);

// 各オブジェクトの情報を送信
for (auto& obj : gameObjects) {
    HapiColi::ObjectData data;
    data.id = obj->GetId();
    data.type = obj->GetType();
    
    data.position = HapiColi::Vector3(obj->pos.x, obj->pos.y, obj->pos.z);
    data.velocity = HapiColi::Vector3(obj->vel.x, obj->vel.y, obj->vel.z);
    
    data.collision.isColliding = obj->IsColliding();
    if (obj->IsColliding()) {
        data.collision.collidedWithId = obj->GetCollidedObjectId();
    }
    
    HapiColi::HapiColi::GetInstance().RecordObject(data);
}

// フレームの最後
HapiColi::HapiColi::GetInstance().EndFrame();
```

### 17.1.3. デバッグ描画（対象オブジェクトのハイライト）
ゲームの描画ループ（ImGui描画中または前）でビュープロジェクション行列を渡すことで、現在UIで選択されているSubjectやTargetのワイヤーフレーム等を画面上にハイライト表示できる。

```cpp
// 行列の先頭ポインタ（float[16]）を渡す
HapiColi::HapiColi::GetInstance().DrawDebug3D(&viewProjMatrix.m[0][0]);
```

## 17.2. ツール画面（ImGui）の使い方

`Update()` を呼び出すと表示される「HapiColi Debugger」ウィンドウ上での操作手順。

### 17.2.1. Recorder（記録機能）
* **Start Recording / Stop Recording**: データの記録を開始・停止する。
* **Clear Data**: 記録したデータをリセットする。
* **Add Target ID**: 記録対象のオブジェクトIDを指定して絞り込む（未指定の場合は全オブジェクトを記録）。

### 17.2.2. Analyzer（解析機能）
期待する動作（ルール）を定義して、記録したデータの成否を判定する。
* **Subject ID**: テスト対象のオブジェクトID（例: `Player`）
* **Expected Hit / Expected No Hit**: 期待する動作（当たるべきか、当たらないべきか）
* **Target ID**: 相手のオブジェクトID（例: `Ground` または `Wall`）
* **Add Rule**: ルールを追加する。
* **Run Analysis**: 追加したルールに基づいて記録データを解析し、結果（Happy/Unhappyの回数と成功率）を表示する。

### 17.2.3. Optimizer（改善提案機能）
* **Generate Suggestions**: 解析結果で「Unhappy」が発生した場合、原因を推測して最適化の提案を表示する。（例: 「コライダーを縮小してください」「Raycastを導入してください」など）

### 17.2.4. File I/O
* **Save Log**: 現在のテスト結果（サマリー、衝突期間、Unhappyフレームの詳細、オブジェクト間の距離等）を人間が読みやすいMarkdownの表形式で `HapiColi/Log/YYYYMMDD_HHMMSS_Report.md` として書き出す。
